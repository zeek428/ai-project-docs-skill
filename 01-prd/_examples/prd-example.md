# PRD 示例：用户管理

> 本文件为完整示例，仅供参考，不复制到实际项目。

## 背景

当前系统缺少用户管理功能，无法支持用户注册、登录、信息管理等基础能力，影响后续业务功能开发。

## 目标用户

- 使用 APP 的终端用户
- 年龄：18-45岁
- 场景：需要快速登录使用服务，不希望记忆复杂密码

## 用户故事

```
作为 用户
我希望 使用手机验证码快速登录
以便 无需记忆密码即可使用服务
```

### 详细场景

1. **场景一：老用户登录**
   - 用户打开 APP，输入手机号，获取验证码，输入后登录成功

2. **场景二：新用户注册**
   - 新用户输入手机号，获取验证码，输入后自动注册并登录

3. **场景三：微信授权登录**
   - 用户点击"微信登录"，授权后自动登录

## 功能范围

### In Scope
- 手机号验证码登录/注册
- 微信授权登录
- 用户信息查看和修改
- 账号退出登录

### Out of Scope
- 密码登录
- 第三方登录（微信除外）
- 用户注销账号

## 功能详情

### 功能点 1：手机号验证码登录

**描述**: 用户输入手机号，通过短信验证码完成登录/注册

**规则**:
- 手机号必须为11位，格式校验
- 验证码6位数字，5分钟有效期
- 同一手机号60秒内只能获取一次验证码
- 新用户自动注册

**异常处理**:
- 手机号格式错误：提示"请输入正确的手机号"
- 验证码错误：提示"验证码错误"
- 验证码过期：提示"验证码已过期，请重新获取"
- 发送频繁：提示"发送过于频繁，请稍后再试"

### 功能点 2：微信授权登录

**描述**: 用户通过微信授权快速登录

**规则**:
- 首次授权自动创建账号
- 后续授权直接登录
- 获取用户昵称和头像

**异常处理**:
- 未安装微信：隐藏微信登录入口
- 授权失败：提示"授权失败，请重试"

## 交互设计

见 [prototype/index.html](./prototype/index.html)

### 页面流程
```
启动页 → 登录页 → 验证码页 → 首页
          ↓
       微信授权 → 首页
```

### 关键交互
1. 登录页自动聚焦手机号输入框
2. 验证码自动填充（iOS）
3. 登录成功后跳转来源页或首页

## 验收标准

- [ ] AC1: 用户可使用手机号验证码登录成功
- [ ] AC2: 新用户自动注册并登录
- [ ] AC3: 用户可使用微信授权登录
- [ ] AC4: 用户可查看和修改个人信息
- [ ] AC5: 登录状态在7天内保持有效

## 非功能需求

| 类型 | 要求 |
|------|------|
| 性能 | 登录接口响应时间 < 500ms |
| 安全 | 验证码加密存储，Token 有效期7天 |
| 可用性 | 登录成功率 > 99.9% |
| 兼容性 | iOS 12+、Android 8.0+ |
