# 故障响应

## 故障等级定义

| 级别 | 定义 | 响应时间 | 升级时间 |
|------|------|----------|----------|
| P0 | 核心服务不可用，影响全部用户 | 5 分钟 | 15 分钟 |
| P1 | 主要功能异常，影响部分用户 | 15 分钟 | 30 分钟 |
| P2 | 次要功能异常，影响少数用户 | 30 分钟 | 2 小时 |
| P3 | 小问题，不影响用户使用 | 2 小时 | 24 小时 |

## 常见故障处理

### 服务不可用

**现象**: 服务无响应或超时

**排查步骤**:
1. 检查服务状态: `kubectl get pods`
2. 检查日志: `kubectl logs <pod>`
3. 检查资源: CPU/内存/磁盘
4. 检查依赖服务

**快速恢复**:
```bash
# 重启服务
kubectl rollout restart deployment/app

# 扩容
kubectl scale deployment/app --replicas=5
```

### 数据库故障

**现象**: 数据库连接失败或慢查询

**排查步骤**:
1. 检查数据库状态
2. 检查连接数
3. 检查慢查询日志
4. 检查磁盘空间

**快速恢复**:
```sql
-- 终止慢查询
SHOW PROCESSLIST;
KILL <id>;
```

### 缓存故障

**现象**: 缓存失效导致数据库压力增大

**排查步骤**:
1. 检查 Redis 状态
2. 检查内存使用
3. 检查连接数

**快速恢复**:
```bash
# 重启 Redis
kubectl rollout restart deployment/redis
```
