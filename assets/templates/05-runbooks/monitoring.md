## 监控指标

### 基础指标
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| CPU 使用率 | 服务 CPU 占用 | > 80% |
| 内存使用率 | 服务内存占用 | > 85% |
| 磁盘使用率 | 磁盘空间占用 | > 90% |
| 网络流量 | 入出流量 | 异常波动 |

### 应用指标
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| 请求量 QPS | 每秒请求数 | 异常波动 |
| 响应时间 P99 | 99 分位响应时间 | > 1s |
| 错误率 | 请求错误比例 | > 1% |
| 慢查询 | 数据库慢查询 | > 500ms |

### 业务指标
| 指标 | 说明 | 告警阈值 |
|------|------|----------|
| 订单量 | 每小时订单数 | 异常下降 |
| 支付成功率 | 支付成功比例 | < 95% |
| 活跃用户 | 在线用户数 | 异常波动 |

## 告警配置

### 告警级别
| 级别 | 说明 | 通知方式 |
|------|------|----------|
| P0 | 严重故障，服务不可用 | 电话 + 短信 + 钉钉 |
| P1 | 影响部分功能 | 短信 + 钉钉 |
| P2 | 需关注，暂不影响用户 | 钉钉 |
| P3 | 提示性信息 | 邮件 |

### 告警规则
```yaml
# 示例告警规则
groups:
  - name: app-alerts
    rules:
      - alert: HighErrorRate
        expr: rate(http_errors_total[5m]) > 0.01
        for: 5m
        labels:
          severity: P1
        annotations:
          summary: "错误率超过1%"
```
